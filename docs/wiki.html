sysinfo-cli 文档

项目介绍

sysinfo-cli 是一个用 Bash 写的 Linux 服务器状态面板。SSH 登录就能看到 CPU、内存、磁盘、网络等信息，不用额外装监控软件，也不跑后台进程。支持 Debian、Ubuntu、RHEL、Fedora、Alpine、Arch、openSUSE 等发行版（不支持 macOS）。

🔐

登录即显示

bash 和 zsh 登录后自动弹出状态面板

📊

实时监控

运行 sysinfo 进入刷新式面板，不闪屏

📈

流量统计

按月统计流量，快到上限时自动限速

🔔

手机推送

通过 Bark 把告警推送到手机

适合谁用？ 个人 VPS、家用服务器、小团队运维——任何 Linux 机器（Debian/Ubuntu/RHEL/Fedora/Alpine/Arch 等），想登录时快速看一眼状态的人。

安装

三种方式任选，安装脚本会自动识别包管理器、装好依赖（yq、iproute）并配置登录横幅。

系统要求

项目说明
操作系统Linux（bash）；不支持 macOS
发行版Debian、Ubuntu、RHEL、CentOS、Rocky、Alma、Fedora、Alpine、Arch、openSUSE、Gentoo 等
包管理器apt、dnf、yum、apk、pacman、zypper、emerge（自动检测）
tc / ipiproute2（Debian/Alpine/Arch）或 iproute（RHEL/Fedora）
yqmikefarah 版，按 CPU 架构下载（amd64 / arm64 / arm）
权限安装与限速需要 root 或 sudo
Alpine 提示 默认 shell 是 ash，需先安装 bash：apk add bash。安装脚本会尝试用 apk 安装 iproute2。

一键安装

bash <(curl -sSL baixiaosheng.de/sysinfo)

从 GitHub 安装

curl -sSL https://raw.githubusercontent.com/baixiaoshengofficial/sysinfo-cli/main/install.sh | bash

克隆后安装

git clone https://github.com/baixiaoshengofficial/sysinfo-cli.git
cd sysinfo-cli
sudo ./install.sh

安装参数

参数说明
--lang zh|en指定界面语言，不写则按系统语言自动判断
--overwrite-config用默认模板覆盖已有配置（旧配置会自动备份）
--uninstall卸载 sysinfo-cli
小提示 重复执行 install.sh 不会覆盖你改过的 config.yaml。只有加 --overwrite-config 才会重置。

升级与卸载

升级

cd sysinfo-cli && git pull && sudo ./install.sh

想同时把配置也恢复成最新模板：

sudo ./install.sh --overwrite-config

卸载

sudo ./uninstall.sh

会删除所有脚本、配置文件、状态文件和 zsh 登录 hook。

常用命令

命令做什么
sysinfo打开实时面板（默认 1 秒刷新）
sysinfo 2每 2 秒刷新一次
sysinfo -r重新加载 /etc/sysinfo/config.yaml
sysinfo -c 文件.yaml从指定配置文件应用设置
sysinfo --reset-traffic清零本月流量统计
sysinfo --clear-nat清除 NAT 端口映射
sysinfo --notify-test发一条测试推送，检查 Bark 是否配好
sysinfo --notify-check检查告警规则（适合放 cron 定时跑）
注意 配置只能通过 YAML 文件修改（-c / -r），旧的 --nat、--traffic 等命令行参数已经不用了。

配置文件

配置文件在 /etc/sysinfo/config.yaml，改完后执行 sudo sysinfo -r 生效。

完整示例

network:
  interface: ""                    # 留空自动检测网卡
  force_gateway_throttle: false    # 网关模式下是否强制限速

nat:
  enabled: false
  mappings:
    - "8080:80"                    # 公网端口:内网端口
    - "9000:3000"

traffic:
  enabled: true
  limit: "1T"                      # 1T / 500G / 100M / UNLIMITED
  reset_day: 1                     # 每月几号重置流量
  mode: "both"                     # upload / download / both

throttle:
  enabled: true
  threshold: 95                    # 用到 95% 开始限速
  rate: "10mbps"                   # 限速到 10 Mbps

display:
  language: "auto"                 # auto / zh / en
  refresh_interval: 1
  show_traffic: true
  show_nat: true
  show_throttle: true

notify:
  enabled: false
  bark:
    url: "https://api.day.app"
    key: ""
  cooldown: 1800
  rules:
    cpu:      { enabled: false, threshold: 90 }
    net:      { enabled: false, threshold: 90 }
    nic:      { enabled: false, threshold: 80, mode: both, upload_rate: 0, download_rate: 0 }
    throttle: { enabled: false }
    disk:     { enabled: false, threshold: 90, paths: [] }
关于 NAT NAT 配置只在面板上展示端口映射关系，不会帮你改 iptables 或 nftables 规则。

流量统计与限速

流量怎么统计

  • 从 /proc/net/dev 读取网卡收发字节数，按月累计
  • 数据存在 /etc/sysinfo-traffic.json，到 reset_day 那天自动清零
  • mode 可以只统计上传、只统计下载，或者两个方向都算

自动限速怎么工作

当本月流量用到 threshold 设定的百分比后，sysinfo-cli 会用 Linux tc 命令把网速限制到 rate 指定的值。

流量达到 95%
→
tc 限速到 10 Mbps
→
面板显示「限速中」
网关模式 如果服务器开了 IP 转发（做路由/网关），默认不会限速，怕影响下游设备。确实需要的话，在配置里设 force_gateway_throttle: true。

bash scripts/test_throttle.sh

推送通知

通过 Bark 把告警推送到 iPhone。默认全部关闭，需要在 config.yaml 里手动开启。

怎么开启

  1. 在 iPhone 上安装 Bark App，拿到推送 Key
  2. 编辑 config.yaml，填上 bark.key，把 notify.enabled 改成 true
  3. 开启你需要的规则（比如 cpu.enabled: true）
  4. 执行 sudo sysinfo -r 应用配置
  5. 运行 sysinfo --notify-test 测试能不能收到

五种告警规则

规则什么时候推送
cpuCPU 使用率超过阈值
net本月流量用到配额的百分之几
nic网速达到带宽的百分之几（可分别监控上传/下载）
throttle自动限速被触发时
disk某个磁盘分区使用率超过阈值
不会刷屏 告警是「边沿触发」的——只在状态变化时推送，同一条规则有 cooldown 冷却时间（默认 30 分钟）。

定时检查（cron）

# 每 5 分钟检查一次告警规则
*/5 * * * * /usr/local/bin/sysinfo --notify-check

切换语言

在 config.yaml 里改 display.language：

display:
  language: "zh"    # auto / zh / en
  • auto — 跟随系统语言
  • zh — 中文界面
  • en — 英文界面

改完运行 sysinfo 立即生效；运行 sudo sysinfo -r 可以让 SSH 登录横幅也一起切换。

SSH 登录横幅

安装后，每次 SSH 登录会自动显示系统状态面板，不用手动敲 sysinfo。

  • bash 用户：通过 /etc/profile.d/ 加载
  • zsh 用户：通过 /etc/zsh/zprofile 加载
  • scp、sftp、ssh host command 这类非交互操作不会显示
  • 同一次登录只显示一次

文件路径

路径用途
/usr/local/bin/sysinfo命令入口
/etc/sysinfo/config.yaml主配置文件
/etc/sysinfo-traffic.json流量计数和限速状态
/etc/sysinfo-lang持久化的界面语言
/var/tmp/sysinfo-notify-state-<用户>推送告警的去重状态

常见问题

支持哪些操作系统？

仅支持 Linux（需 bash）。常见发行版：Debian、Ubuntu、RHEL、CentOS、Rocky、Alma、Fedora、Alpine、Arch、openSUSE、Gentoo。安装脚本自动识别 apt/dnf/yum/apk/pacman/zypper/emerge。macOS 不支持（依赖 Linux tc、IFB 等）。

yq 报 Exec format error？

通常是 ARM 服务器上装了 amd64 的 yq。删除后重装：sudo rm -f /usr/local/bin/yq，再执行 install.sh（会按 uname -m 下载 yq_linux_arm64 等）。验证：yq --version 和 yq eval '.display.language' /etc/sysinfo/config.yaml。

SSH 登录没看到面板？

确认是正常登录（不是 scp 传文件）。zsh 用户如果还是没有，重新执行一次 sudo ./install.sh 修复 hook。

改了配置但没效果？

普通用户会优先读 ~/.config/sysinfo/config.yaml，如果存在就会覆盖系统配置。删掉它，或者指定 SYSINFO_CONFIG=/etc/sysinfo/config.yaml sysinfo。

推送收不到？

先跑 sysinfo --notify-test 看通道通不通。确认 notify.enabled 是 true、bark.key 填对了。测试时可以把 threshold 临时改成 0。

限速没生效？

网关模式（开了 IP 转发）默认不限速。需要 tc 命令和 root 权限。跑 bash scripts/test_throttle.sh 看诊断结果。
